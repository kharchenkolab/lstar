# R API (reference)

Package `lstar` (under `R/`). Binds the header-only C++ core `libstar` via **cpp11**. An
`lstar_dataset` is an S3 list with `$axes` and `$fields` (each field's `$values` is a base
vector/matrix or a `Matrix` sparse matrix).

## Core IO

```r
library(lstar)
ds  <- lstar_read(path)                 # -> lstar_dataset; reads multi-chunk + gzip/zlib stores
lstar_write(ds, path)                   # -> path (invisibly)
field_value(ds, name)                   # a field's $values
print(ds)                               # summary of axes + fields
```

An `lstar_dataset` you build by hand:
```r
ds <- list(kind = "sample", axes = list(), fields = list())
ds$axes$cells  <- list(labels = paste0("c", 1:N), origin = "observed", role = "observation")
ds$axes$genes  <- list(labels = gene_ids,          origin = "observed", role = "feature")
ds$fields$counts <- list(role = "measure", span = c("cells","genes"), state = "raw",
                         values = dgCMatrix_cells_x_genes)   # CSC dgCMatrix maps to L* csc
ds$fields$leiden <- list(role = "label", span = "cells", values = cluster_factor)
class(ds) <- "lstar_dataset"
```
Orientation: L★ measures are `(cells × genes)`; Seurat/SCE assays are `(genes × cells)` — the
profiles transpose with `Matrix::t`. A `dgCMatrix` (CSC) maps directly to L★ `csc`.

## Profiles

```r
# Seurat (legacy v2 pre-Assay 'seurat' class, v3/v4 Assay, v5 Assay5; a split assay -> an L* collection)
so  <- write_seurat(ds)                 # L* -> Seurat (measures->layers, embeddings->DimReduc, fields->meta.data)
ds  <- read_seurat(so, assay = "RNA")   # Seurat -> L*; records SeuratObject@<v> + assay@<v3|v5>

# SingleCellExperiment
sce <- write_sce(ds)                    # measures->assays, embeddings->reducedDims, fields->colData
ds  <- read_sce(sce)

# Conos (a collection of pagoda2 samples) -> L* collection
ds  <- write_conos(conos_obj, clustering = NULL)   # samples axis + per-sample axes/counts/pca +
                                                   # union cells + sample label + joint embedding/
                                                   # clusters + graph(relation). Records conos@/pagoda2@.
```

### Collections
- Seurat v5: `obj[["RNA"]] <- split(obj[["RNA"]], f = obj$sample)` makes a split assay; `read_seurat`
  reads it as `kind="collection"` (per-sample `cells.<s>` axes + `counts.<s>` + `samples` axis + a
  `sample` design label).
- Conos: `write_conos` ingests the whole object as a collection (see `reference/model.md`).

## Version recognition (graceful)
- Seurat: `Assay` (v3/v4, fixed counts/data/scale.data slots) vs `Assay5` (v5, layers); a
  `GetAssayData` fallback covers SeuratObject < 5 (no `Layers()` API). Recorded in `ds$profiles`.
- pagoda2: `getRawCounts()` accessor vs the legacy `$counts` slot (recorded `pagoda2-accessor`/`-slot`).
- Anything a profile can't represent goes to `ds$dropped` rather than erroring.

## Packaging (CRAN)
- `R CMD check --as-cran` is clean (0 ERROR/0 WARNING; NOTEs are new-submission/size/timestamp).
- `man/*.Rd` are roxygen-generated (`Roxygen: list(markdown = TRUE)`); regenerate with
  `roxygen2::roxygenise(roclets="rd")`. The cpp11 glue (`src/cpp11.cpp`, `R/cpp11.R`) is generated by
  `cpp11::cpp_register()` from `[[cpp11::register]]` attributes and **committed** (users don't need
  `decor`).
- `src/Makevars`: `CXX_STD=CXX17`, `PKG_CPPFLAGS=-I../inst/include -DLSTAR_HAVE_ZLIB`, `PKG_LIBS=-lz`.
  The C++ core is vendored under `R/inst/include/` (`lstar/lstar.hpp`, `nlohmann/json.hpp`).
- Install dev build: `R CMD INSTALL --preclean R` (preclean matters — R's make has no header-dep
  tracking, so a changed vendored header is otherwise relinked stale).
